# MaGiBu

**MaGiBu** (Methylation-based Guided Builder) is a modular R framework for building methylation-based classification models. It is platform-agnostic, supporting both Illumina methylation arrays (450K, EPIC, EPICv2) and Oxford Nanopore sequencing data.

## Overview

This tutorial covers:

0. **How to install MaGiBu** Spoiler: you don't need  
1. **Building a reference model** from a training dataset
2. **Projecting new samples** onto the reference model for classification

## Functions loading
To load the functions into your R environment, navigate to the directory where you saved the files and run the following command:

```r
sapply(list.files(), source)
```

## Part 1: Building the Reference Model

### Prerequisites

To perform predictions, you first need to create a reference model from a training dataset. In this tutorial, we use the Capper et al. dataset (PMID: 29539639), available from GEO accessions GSE90496 and GSE223546, with sample annotations provided in this repository at `CapperAnnotations/samples.clinics.3905.tsv`.

### Step 1: Process IDAT Files

After downloading the IDAT files, process them to extract beta values:
```r
process_methylation_data(
 path.to.idat = "path/to/idats",
 array.type = "EPICv2",
 save.path = "path/to/tsv/reference_samples",
 n.cores = 60
)
```

> ⚠️ **Important**: Ensure the output directory (`path/to/tsv/reference_samples`) contains **only** the methylation TSV files generated by this function.

### Step 2: Generate the UMAP Training Model
```r
umaps <- generate_umap_training_model(
 methylation.path = "path/to/tsv/reference_samples",
 metadata_file = "CapperAnnotations/samples.clinics.3905.tsv",
 save.path = "path/to/reference",
 name = "capper.umap",
 n_top_vars = 50000,
 pca_components = 50,
 n_neighbors = 15,
 n_cores = 60
)
```

The `save.path` parameter saves the model components for future use. Pre-built model files based on the Capper et al. dataset are available in the `Models/` directory.

## Part 2: Projecting New Samples

### Step 1-A: Process New Sample IDAT Files

For this example, we use samples from Mattei et al. (GSE319846):
```r
process_methylation_data(
 path.to.idat = "path/to/new_idats",
 array.type = "EPICv2",
 save.path = "path/to/tsv/new_samples",
 n.cores = 60
)
```

### Step 10-B: Process New Sample Nanopore BAM Files

To process Nanopore bam files it is necessary to run modkit with the following command:

```
REF=hg38.fasta # your fasta reference It must match the one used for the bam alignment.
THN=10
modkit extract full --mapped-only --threads ${THN} --cpg --reference ${REF} dorado.output.bam modkit.output.tsv
```

Then Run the PoreMeth2 script ModkitResorter.sh. 
You can retrieve ModkitResorter.sh and ParseModkit.pl (required) from [https://github.com/Lab-CoMBINE/PoreMeth2](https://github.com/Lab-CoMBINE/PoreMeth2/tree/master/AdditionalScripts

```
sh ModkitResorter.sh modkit.output.tsv
mv *sorted.entropy* /path/to/tsv/new_samples

```
It will generate the TSV files used in the project_new_samples function 

> ⚠️ **Important**: Again, ensure the output directory contains **only** methylation TSV files.

### Step 2: Project Samples onto the Reference Model
```r
projected <- project_new_samples(
 training.metadata = "path/to/reference/capper.umap_metadata_result.tsv",
 model.components = "path/to/reference/capper.umap_model.components.rds",
 uwot.model = "path/to/reference/capper.umap_umap_model.uwot",
 path.new.tsv = "path/to/tsv/new_samples",
 save.path = "path/to/new_projection"
)

# Alternatively, you can pass the umaps object directly if still in memory
```

### Step 3: Visualize Results

Create a sample ID vector from your processed files:
```r
my_samples <- gsub("\\.tsv$", "", list.files("path/to/tsv/new_samples"))
```

> **Note**: The order of sample names in this vector determines the order of results.

#### Generate UMAP Plot
```r
plot_umap_zoom_combined(
 metadata.in = projected$metadata,
 output.path = "path/to/plots",
 name = "tutorial_files",
 sample_id = my_samples
)
```

#### Examine Classification Distances
```r
get_top_3_distances(
 projected$metadata,
 sample_ids = my_samples
)
```

This function returns the three nearest reference classes for each sample, along with their distances, useful for assessing classification confidence.
